STAGE 1: HTML FOUNDATIONS (Day 1–2)
🎯 What HTML Really Is

HTML = HyperText Markup Language

It is NOT programming.
It is a structure language used to tell the browser:

“This is a heading, this is a paragraph, this is an image…”

🧠 How the Web Works (Simplified)

You write HTML

Browser reads it

Browser renders UI

🧱 Basic Structure of Every HTML Page
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>

  <body>
    <h1>Hello World</h1>
    <p>This is my first page</p>
  </body>
</html>
🔍 Breakdown:

<!DOCTYPE html> → tells browser it's HTML5

<html> → root of page

<head> → settings (not visible)

<body> → visible content

What is an HTML Element?

An HTML element is everything from the opening tag to the closing tag:

<p>This is a paragraph</p>

👉 This whole thing is ONE element.

🧱 Structure of an Element
<tagname attribute="value">Content</tagname>

Example:

<a href="https://example.com">Visit</a>

<a> → opening tag

href → attribute

Visit → content

</a> → closing tag

🔍 TYPES OF HTML ELEMENTS

There are 3 main types you must understand:

1️⃣ Normal Elements (Container Elements)

They have:

Opening tag

Content

Closing tag

Examples:

<div>Hello</div>
<p>Text</p>
<h1>Title</h1>
2️⃣ Empty Elements (Self-Closing Elements)

These DO NOT have closing tags
They also DO NOT contain content

⚡ Common Empty Elements
<br>      <!-- line break -->
<hr>      <!-- horizontal line -->
<img>     <!-- image -->
<input>   <!-- form input -->
<meta>    <!-- metadata -->
<link>    <!-- external resources -->

Example:

<img src="image.jpg" alt="photo">
<br>
<hr>
❗ Important Rule:

Empty elements don’t wrap anything

❌ Wrong:

<br>Text</br>

✅ Correct:

<br>
3️⃣ Nested Elements

Elements inside other elements:

<div>
  <p>Hello <b>World</b></p>
</div>

👉 This is how real layouts are built.

📦 BLOCK vs INLINE ELEMENTS (VERY IMPORTANT)

This is one of the most important concepts in HTML.

🟥 BLOCK ELEMENTS
🔥 Behavior:

Take full width

Start on a new line

Stack vertically

✅ Examples of Block Elements:
<div></div>
<p></p>
<h1></h1> - <h6></h6>
<section></section>
<header></header>
<footer></footer>
<ul></ul>
<li></li>
🧠 Example:
<p>First</p>
<p>Second</p>

👉 Output:

First
Second
🟦 INLINE ELEMENTS
🔥 Behavior:

Take only needed width

Stay on the same line

Do NOT break line

✅ Examples of Inline Elements:
<span></span>
<a></a>
<img>
<b></b>
<i></i>
<strong></strong>
<em></em>
🧠 Example:
<span>Hello</span>
<span>World</span>

👉 Output:

Hello World
⚖️ BLOCK vs INLINE (Clear Comparison)
Feature	Block Element	Inline Element
Width	Full width	Content width
Line Break	Yes	No
Layout Use	Structure	Text styling
Examples	div, p, h1	span, a, b

⚠️ IMPORTANT RULES
❌ Inline inside Block → ✅ OK
<p>Hello <span>World</span></p>
❌ Block inside Inline → 🚫 BAD PRACTICE
<span><div>Wrong</div></span>

PART 1 — What Forms REALLY Are (Senior Perspective)

A form is a contract for user data collection.

When a user clicks submit, the browser creates a HTTP request containing:

URL
Method (GET / POST)
Headers
Body (form data)

Think of forms as a frontend API client built into the browser.

Flow:

User → Form → HTTP Request → Backend → DB → Response → UI

PART 2 — The <form> Element

Basic skeleton:

<form action="/signup" method="POST">
  <input name="email">
  <button type="submit">Submit</button>
</form>
Core Attributes (VERY IMPORTANT)
Attribute	What it does	Real usage
action	Where data goes	Backend endpoint
method	GET or POST	Defines request type
enctype	Encoding type	File uploads
autocomplete	Browser autofill	UX improvement
novalidate	Disable HTML validation	When using JS validation
target	Where response opens	Rare (iframe/blank)
action
<form action="/login">

Where the browser sends the request.

If omitted → submits to current page URL.

method (GET vs POST)
GET → retrieving data
<form method="GET">

Data goes in URL:

/search?q=shoes&page=2

Used for:

Search bars
Filters
Pagination
Shareable URLs

POST → sending sensitive data
<form method="POST">

Data goes in request body.

Used for:

Login
Signup
Payments
File uploads

Rule of thumb:

GET = fetch data
POST = change data

enctype (super important for files)

Default:

application/x-www-form-urlencoded

For file uploads:

<form enctype="multipart/form-data">

Without this → file uploads will fail ❌

Always use labels

<input type="email">

<input id="email" type="email">

Always validate on BOTH sides

Backend validation = Security 🔒

Never trust client data.

Use GET for search/filter forms

PART 1 — The <video> Element

Basic example you gave:

<video controls src="video.mp4"></video>

This works, but in real projects it’s not production-ready.

Let’s build it properly.

Proper Production Video Structure
<video controls width="600" poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support video.
</video>
Why multiple <source> files?

Different browsers support different codecs:

Format	Why used
MP4 (H.264)	Works everywhere
WebM	Smaller size, modern browsers
Ogg	Rare fallback

Browser automatically picks the first supported format.

This is called progressive enhancement.

Video Attributes (Important)
controls
<video controls>

Adds play/pause/volume/fullscreen UI.

Without this → user cannot control playback.

width / height
<video width="640" height="360">

Prevents layout shift (important for performance).

autoplay ⚠️
<video autoplay muted>

Browsers block autoplay WITH sound.

So autoplay videos must be muted.

Real usage:

Hero background videos
Landing pages
loop
<video loop>

Used for:

Background animations
Product demos
muted
<video muted>

Often used with autoplay.

poster (thumbnail)
<video poster="preview.jpg">

Used for:

YouTube-like preview image
Better UX before playback
preload (performance control)
<video preload="none">

Options:

Value	Meaning
none	Don’t download video until play
metadata	Load duration only
auto	Load entire video

Best practice:

preload="metadata"

PART 3 — Subtitles & Captions (Very Important)

This is where senior devs stand out.

Use <track>:

<video controls>
  <source src="movie.mp4" type="video/mp4">

  <track src="subtitles_en.vtt"
         kind="subtitles"
         srclang="en"
         label="English"
         default>
</video>
Why subtitles matter:
Accessibility ♿
SEO
Silent viewing on mobile
Legal compliance in some countries
Track kinds
kind	Usage
subtitles	Translation
captions	Includes sound effects
chapters	Video navigation
descriptions	Screen reader narration
PART 4 — Supported Formats Cheat Sheet
Video formats
Format	Extension	Codec
MP4	.mp4	H.264
WebM	.webm	VP8/VP9
Ogg	.ogv	Theora
Audio formats
Format	Extension

Production rule:
Always provide MP4 + WebM for video
Always provide MP3 + OGG for audio

PART 5 — Accessibility Best Practices

Always provide fallback text:

<video controls>
  <source src="video.mp4">
  Your browser does not support HTML5 video.
</video>

Why?
Older browsers or screen readers.

PART 6 — Real World Example (Media Section)

This looks like a real website section:

  <video controls width="700" poster="demo.jpg" preload="metadata">

    <source src="demo.webm" type="video/webm">

           kind="subtitles"
           srclang="en"
           label="English"
           default>
  </video>

  <h2>Listen to our Podcast</h2>

  <audio controls preload="metadata">
    <source src="podcast.mp3" type="audio/mpeg">
    <source src="podcast.ogg" type="audio/ogg">
  </audio>
</section>

This is production-grade.
